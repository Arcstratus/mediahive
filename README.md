# Mediahive：個人媒體素材管理系統

## 這是什麼

Mediahive 是一個本地端的媒體素材管理工具，透過瀏覽器介面操作。
主要用來管理部落格、網站、遊戲開發過程中累積的大量圖片與影片素材。

這是個人使用的開源專案，不是 SaaS，不需要帳號系統或多人協作。
它跑在本機上，前端透過瀏覽器存取，後端管理檔案與資料庫。

## 為什麼需要它

素材一多，資料夾就失控。
找不到東西、忘記哪張圖用過、重複檔案散落各處。
作業系統的檔案總管不支援標籤、不能批次處理、不能預覽影片截圖。
我需要一個專門為「管理大量媒體素材」設計的工具，而且要能按照自己的需求持續擴充。

## 核心能力

- **匯入與整理**：上傳檔案後自動以 hash 重新命名，天然去重，統一存放
- **瀏覽與檢索**：縮圖網格快速瀏覽大量素材，支援分頁、過濾、排序
- **標籤系統**：對素材打標籤，透過標籤組合過濾，快速定位需要的檔案
- **單檔檢視**：全尺寸檢視圖片、播放影片，鍵盤左右鍵切換上下一個檔案
- **檔案操作**：改路徑、重新命名、刪除、批次處理
- **圖片轉檔**：格式轉換（如 BMP 轉 WebP），背景處理
- **影片處理**：自訂影片預覽圖、影片轉檔、截圖
- **M3U8 支援**：輸入 URL 播放 m3u8 串流、下載 m3u8 存為本地檔案
- **NAS 整合**（未來）：掛載外部儲存路徑，管理 NAS 上的素材

## 技術選型

- **後端**：Python，FastAPI，SQLite（未來可能遷移至 PostgreSQL）
- **前端**：Nuxt 3，Nuxt UI，Vue 3 Composition API，TypeScript
- **套件管理**：Python 用 uv，Node 用 pnpm
- **外部依賴**：FFmpeg（處理影片縮圖、轉檔等，需系統預先安裝）
- **資料庫遷移**：Alembic

## 設計原則

- **Monorepo**：前後端放在同一個 repo，簡化管理
- **漸進式開發**：功能按階段疊加，每個階段獨立可用，不依賴後續階段
- **本地優先**：所有資料存在本機，不依賴雲端服務
- **瀏覽器介面**：跨平台（Windows、macOS），不需要安裝桌面應用
- **不做身份驗證**：個人工具，本地網路使用，不需要登入機制
